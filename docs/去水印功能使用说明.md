# 去水印功能使用说明

## 概述

该功能适用于简单水印处理且处理效果很粗糙,如ai标识

去水印功能位于 **设置 > 工具 > 去水印**（快捷键 `Ctrl+M`），支持对单张或批量图片进行水印去除处理。提供三种处理算法和三种区域绘制方式，用户手动框选水印区域后即可一键处理。

## 绘制模式

工具栏「绘制」下拉框提供三种区域选择方式，可混合使用：

### 1. 矩形框选（默认）

| 操作 | 说明 |
|------|------|
| **左键拖拽** | 拖拽画矩形框，松开后添加该区域 |
| **右键点击** | 点击已有区域 → 删除面积最小的重叠区域 |

适用场景：水印是规则的矩形文字或 logo。

### 2. 任意圈选（自由绘制）

| 操作 | 说明 |
|------|------|
| **左键拖拽** | 按住拖拽自由绘制封闭区域，松开后自动闭合 |
| **右键点击** | 删除已有区域 |

适用场景：水印形状不规则，需要沿水印轮廓精确描绘。

> 采样点过少（< 10 个）或闭合后顶点不足 3 个的绘制会被忽略（视为误触）。

### 3. 多点连线（多边形）

| 操作 | 说明 |
|------|------|
| **左键单击** | 逐个放置顶点 |
| **鼠标移动** | 实时预览从最后顶点到光标的连线 |
| **双击** | 完成多边形（自动去除双击产生的重复点） |
| **Enter** | 完成当前多边形 |
| **Escape** | 取消正在绘制的多边形 |
| **右键** | 绘制中 → 取消当前多边形；非绘制中 → 删除已有区域 |

适用场景：水印是多边形形状，或需要精确控制每个顶点位置。

> 每个顶点显示蓝色圆点，方便确认位置。

## 处理模式

工具栏「处理」下拉框提供三种算法：

### 1. 高斯模糊（默认）

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| 模糊核大小 | 51 | 3 ~ 99 | 值越大模糊效果越强 |

**原理**：对全图做高斯模糊，用羽化 mask 将模糊后的选区与原图自然融合。选区边界有 15px 的渐变过渡带，避免硬边界接缝。

**适用场景**：大多数水印，保留背景纹理的同时弱化水印。模糊核越大，水印去除越彻底但背景失真也越大。

### 2. 色块填充

无额外参数。

**原理**：从选区**外围**（外扩环形区域）采样中位色，用该颜色填充选区内部。Fallback 链：外围像素 → 四角采样 → 全图中位色。

**适用场景**：纯色或渐变背景上的水印（如白底黑字水印）。不适合纹理复杂的背景。

### 3. 图像补全（Inpaint）

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| 修复半径 | 5 | 1 ~ 20 | 值越大修复范围越广，但可能引入周围像素失真 |

**原理**：使用 OpenCV TELEA 算法根据周围像素智能填充选区。自动对 mask 做 2px 膨胀以覆盖水印边缘锯齿。

**适用场景**：复杂背景上的水印，效果最好但处理速度较慢。

## 单图模式操作流程

1. 点击 **打开图片** → 选择一张图片（支持 JPG / PNG / BMP / WebP / TIFF）
2. 选择绘制模式，在画布上框选水印区域（可框选多个）
3. 选择处理模式和参数
4. 点击 **处理** → 预览结果
5. 如不满意可点击 **重置** 回到原图重新框选
6. 确认后点击 **保存** → 选择保存路径

> 单图模式下可多次「处理 → 重置 → 重新框选」直到满意。

---

## 批量模式操作流程

1. 点击 **批量模式** 进入批量模式
2. 添加图片：
   - **批量添加**：选择多张图片（可多次点击累加，自动去重）
   - **添加文件夹**：递归扫描文件夹内所有图片
   - **清空列表**：重置批量列表
3. 在左侧文件列表中切换图片，对每张图片分别框选水印区域
4. 选择处理模式和参数
5. 点击 **全部处理** → 后台逐张处理
6. 处理完成后状态栏显示：`成功 X / 失败 Y / 跳过 Z`

### 批量模式注意事项

- 切换图片时自动保存当前图片的选区，返回时自动恢复
- **无选区的图片会被跳过**，不报错
- 输出到源文件旁边的 `nowm/` 子目录（可在配置文件中修改）
  - 例：`C:\photos\a\1.jpg` → `C:\photos\a\nowm\1.jpg`
- 处理中可点击 **取消** 中止，已完成的文件不受影响
- 批量上限默认 200 张（可在配置文件中修改），超过时弹窗确认
- 关闭对话框时如有处理正在进行，会弹窗确认

## 配置项

配置文件 `config.ini` 中的 `[tools]` 节：

```ini
[tools]
watermark_mode = blur           # 默认处理模式：blur / fill / inpaint
watermark_blur_size = 51        # 高斯模糊核大小（奇数）
watermark_inpaint_radius = 5    # 图像补全修复半径
watermark_output_dir = nowm     # 批量输出子目录名（相对于源文件目录）
watermark_max_batch = 200       # 批量处理上限
```

也可通过 **设置 > 编辑配置 > 工具** 标签页在 GUI 中修改。

> `watermark_output_dir` 是相对路径，相对于每个源文件所在目录，不是项目根目录。

## 支持的图片格式

| 格式 | 读取 | 保存 |
|------|------|------|
| JPEG (.jpg / .jpeg) | ✅ | ✅ |
| PNG (.png) | ✅（含 4 通道 alpha） | ✅ |
| BMP (.bmp) | ✅ | ✅ |
| WebP (.webp) | ✅ | ✅ |
| TIFF (.tiff / .tif) | ✅ | ✅ |

PNG 4 通道图片（带透明度）会保留 alpha 通道，处理仅作用于 RGB 通道。

## 中文路径兼容

所有图片读写均使用 `numpy.fromfile` + `cv2.imdecode` / `cv2.imencode` + `numpy.tofile` 方式，完整支持中文路径和中文文件名。

## 技术细节

- **依赖**：`opencv-python-headless>=4.8.0`（已在 `requirements.txt` 中声明）
- **线程安全**：批量处理使用独立 QThread Worker，QRect 等 Qt 类型在传递前转为 tuple
- **内存管理**：批量处理逐张加载、逐张释放，不一次性加载所有图片
- **float32 运算**：高斯模糊模式内部使用 float32 防止 uint8 溢出色偏
- **文件来源**：
  - `src/ui/watermark_dialog.py` — 算法函数 + CanvasWidget + 主对话框
  - `src/ui/watermark_worker.py` — 批量处理 Worker 线程
